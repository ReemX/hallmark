<objective>
Fix UAT tests 6, 7, 14 (CSS surface regression — major) AND tests 3, 6
(drag-region undersized — minor). Five sub-symptoms collapse to ONE root
cause for the CSS gaps, and 3 missing attributes solve the drag-region
gaps — all are pure surface-layer fixes touching one CSS file and 2 TSX
files.

Per the diagnosis (`.planning/debug/settings-wizard-css-surface-regression.md`):

- `src/styles/settings.css` is missing the global `html, body, #root` reset
  that `companion.css` (lines 2-8) and `popup.css` (lines 3-8) both ship.
- Without it, the body has UA-default white background and 8px margin —
  visible "off-white bleed" around the rounded card.
- `.settings-shell` and `.wizard-shell` use `min-height: 100vh` instead of
  `height: 100%`, so `.settings-body { flex: 1; overflow-y: auto }` cannot
  establish a bounded scroll viewport — overflow propagates to body, which
  spawns the native browser scrollbar at the WINDOW edge, OUTSIDE the
  rounded card.
- `.wizard-header` lacks `position: sticky` — scrolls with body content.
- `.skeleton-line` is `height: 36px` with no padding while `.source-row` is
  `min-height: 36px + padding: 8px` — visible row-height jump on rescan.
- `.settings-body` `gap: 32px` between sections is one tier too large for
  580px height with 3 sections + headers + skeleton state.

Per `.planning/debug/drag-region-undersized.md`:

- Tauri 2's `data-tauri-drag-region` is per-element (NOT inherited).
- 3 missing attributes: `CompanionHeader.tsx` title `<div>` (line 6),
  `CompanionHeader.tsx` badge `<div>` (line 8), `Settings.tsx` settings-title
  `<span>` (line 120).

UI-SPEC § "Inheritance note" explicitly required Phase 4 surfaces to inherit
Phase 2's design language; the CSS gap is a regression against that contract.
This plan closes it.

Out of scope: layout density beyond the section-gap tighten (32px to 24px) —
the diagnosis confirmed the perceived "wasted horizontal space" was an
illusion caused by the OS scrollbar stealing real estate; once the scrollbar
moves inside the card, perceived width returns. No HTML/TSX restructure.

Output: One CSS patch (~25 added lines + 4 modified declarations), 3 attribute
additions across 2 TSX files. No backend changes; no Rust compile required.
</objective>
