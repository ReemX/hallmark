    Key v1 design parameters (Claude's discretion within the procedural budget):

    **Standard (popup-standard.wav)** — the workhorse SFX. Targets ~480ms total: 5ms attack + 60ms sustain + 415ms release tail.
    - Layer 1: 880 Hz fundamental @ 0.6 amp (the "ding" body)
    - Layer 2: 1760 Hz first harmonic @ 0.25 amp (warmth)
    - Layer 3: 220 Hz sub-octave @ 0.15 amp (weight)
    - Peak normalize to -3 dBFS
    - Sample rate: 44100, mono

    **Rare (popup-rare.wav)** — richer than standard, 750ms total: longer attack + brighter harmonic content.
    - Layer 1: 1200 Hz fundamental @ 0.55 amp
    - Layer 2: 2400 Hz harmonic @ 0.3 amp
    - Layer 3: 600 Hz sub @ 0.2 amp
    - Layer 4: 3600 Hz "shimmer" overtone @ 0.1 amp
    - Peak normalize to -2 dBFS
    - Sample rate: 44100, mono

    **Completion (popup-100pct.wav)** — celebratory, 1200ms total. Three-note ascending pattern (Do-Mi-Sol).
    - Note 1 (0–400ms): 523 Hz (C5) with 1046/2092 Hz harmonics
    - Note 2 (400–800ms): 659 Hz (E5) with 1318/2636 Hz
    - Note 3 (800–1200ms): 784 Hz (G5) with 1568/3136 Hz
    - Each note has its own attack/release envelope (5ms / 350ms)
    - Peak normalize across all three notes to -2 dBFS
    - Sample rate: 44100, mono

    The existing `synth()` function only handles a single envelope — extend the generator to optionally compose multiple time-shifted layers for the completion track. Acceptable shape:

    ```rust

        offset_ms: u32,

        release_ms: f32,
    ) -> Vec<f32> { ... }

    // In main():
    // popup-100pct.wav as concatenation of three segments at 0/400/800 ms offsets.
    ```

    Or simpler: keep `synth()` single-segment and run it three times into a shared `Vec<f32>` accumulator with offset-based summation, then normalize globally and convert to i16.
