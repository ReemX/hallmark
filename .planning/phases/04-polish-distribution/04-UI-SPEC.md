# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Polish & Distribution.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Inheritance note:** All Phase 4 UI surfaces inherit the Hallmark design language
> established in Phase 2 (companion.css + popup.css). New surfaces — Settings window,
> first-run wizard — must match that language exactly. No new design tokens are
> introduced for Phase 4; this spec confirms the inherited contract and adds
> copywriting + interaction contracts for the new surfaces.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — pure custom CSS, no component library |
| Preset | not applicable |
| Component library | none (Tauri WebView2 + React 19 + Framer Motion 12) |
| Icon library | none — Unicode/SVG inline glyphs only; no third-party icon lib |
| Font | "Inter", "Segoe UI Variable", "Segoe UI", -apple-system, BlinkMacSystemFont, system-ui, sans-serif |

Source: companion.css line 3 + popup.css line 5 (both shipped in Phase 2).

shadcn gate result: `components.json` not present. Tech stack is React + Vite. shadcn
gate skipped because Phase 2 already shipped a custom CSS design system that all new
surfaces must inherit — introducing shadcn at Phase 4 would create a split design system.
Registry safety gate: not applicable.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, rarity-badge inline padding, row margin-bottom |
| sm | 8px | Controls row gap, filter-chip padding, companion-controls padding y |
| md | 16px | Default element padding (companion-header px, list px, popup pill px) |
| lg | 24px | Popup pill padding-right, major element separation |
| xl | 32px | Settings section gap, wizard content padding |
| 2xl | 48px | Empty-state vertical padding (companion-empty: 48px 16px) |
| 3xl | 64px | Not used in Phase 4 surfaces |

Source: companion.css lines 16, 32, 47, 51, 110 (Phase 2 shipped values). Phase 4 new
surfaces (Settings, wizard) apply the same scale — no new tokens.

Exceptions:
- Tray menu: rendered by OS/Tauri native menu subsystem. Spacing is system-controlled.
  Do not apply custom tokens to the tray menu item padding.
- Close button: 28×28px circle (companion.css line 27) — not on 4-point scale but inherited
  from Phase 2's shipped decision; kept as-is for visual continuity.
- Achievement row min-height: 44px (companion.css line 54) — touch-target floor.
- NSIS installer wizard pages: Tauri/NSIS default layout. Do not attempt to apply Hallmark
  tokens to the native installer chrome.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label / UI text | 14px | 600 | 1.2 |
| Meta / caption | 12px | 400 | 1.4 |

Source: companion.css lines 17, 38, 80, 84, 90, 111–112 (Phase 2 shipped values).
Popup uses 14px/600 title + 12px/400 desc (popup.css lines 39–47) — same 14+12 pair.

Rules for Phase 4 new surfaces:

  16px weight 600 line-height 1.2.
- Settings body text and source-status lines: 14px weight 400 line-height 1.5.
- Settings version string and last-checked timestamp: 12px weight 400 line-height 1.4.

- Update modal version badge (e.g. "v1.2.0 available"): 12px weight 400 line-height 1.4,
  rendered in accent color.

No licensed fonts. No web font loading. System font stack only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #111114 | Window background, companion shell, Settings window bg, wizard bg |
| Secondary (30%) | #1C1C21 | Cards, locked achievement rows, filter chips, unselected sort chips |
| Earned row tint | #1A2030 | Earned achievement rows (slightly blue-tinted dark surface) |
| Accent (10%) | rgba(120, 220, 255, 0.85) | Accent-reserved elements only (see below) |
| Destructive | #E05252 | Close button hover state only |
| Popup glass | rgba(12, 12, 16, 0.82) | Popup pill background (translucent, backdrop-blur) |
| Completion purple | rgba(220, 180, 255, 0.40) | 100% completion popup border only |
| Text primary | #F0F0F5 | All body text on dark surfaces |
| Text secondary | rgba(240, 240, 245, 0.55) | Descriptions, meta text, locked achievement titles |
| Text inverse | #FFFFFF | Text inside popup pill (on near-black glass) |

Source: companion.css lines 2–3, 22, 29, 37, 60, 63, 68, 80, 84, 111; popup.css lines
7, 22, 52, 57, 67.

**Accent reserved for (explicit list — no other uses):**
1. Active filter chip border + text (companion controls)
2. Active sort chip border + text (companion controls)
3. Earned-achievement left-bar indicator (4px vertical stripe on row left edge)
4. Session-earned badge in companion header (badge bg + text)
5. Rarity percentage text in popup pill (.popup-rarity)
6. Rare-tier popup icon halo (box-shadow with accent at 0.55 opacity)
7. Update modal version badge text (Phase 4 new — same accent, consistent use)
8. First-run wizard source "found" checkmark (Phase 4 new — ✓ rendered in accent)
9. Settings "Detected sources" found-source indicator (Phase 4 new — same ✓ treatment)

**NOT accent:**
- "Rescan" button: secondary surface #1C1C21 with text primary, no accent border.
- "Get started" / "Continue anyway" buttons: white text on #1C1C21 surface.
  Only the primary wizard CTA ("Get started") gets a subtle accent border to distinguish
  it as the positive path.
- Source "not found" indicators: text-secondary rgba(240,240,245,0.55) with ✗ symbol.
  Never use destructive red for "not found" — this is informational, not an error.

**Completion purple** (rgba(220,180,255,0.40)) is reserved exclusively for the 100%
completion popup border. Do not reuse it in Phase 4 surfaces.

## Surface Specifications

### Settings Window

- Form factor: borderless rounded card. `decorations: false`. Fixed size 520×580px.
  (Source: CONTEXT.md D-10.)
- Background: #111114 (dominant). Border-radius: 12px (match companion card radius).
- Custom drag region: top 48px strip (same height as companion header). `data-tauri-drag-region`.
- Custom close button: 28×28px circle, top-right of header. Background transparent,
  hover #E05252. Same component shape as `.companion-close`.
- Sections separated by a 1px horizontal rule at rgba(255,255,255,0.06)
  (matches companion header border-bottom).
- Rescan button: pill shape, padding 6px 16px, background #1C1C21, text #F0F0F5 14px/400.
  Border: 1px solid rgba(255,255,255,0.10). Hover: border-color rgba(255,255,255,0.20).
- Check for updates button: same pill shape as Rescan.
- GitHub link (About section): accent color rgba(120,220,255,0.85), underline on hover.

### First-Run Wizard Window

- Form factor: borderless rounded card. `decorations: false`. Fixed size 480×560px.
  Centered on tray-icon's monitor on open. (Source: CONTEXT.md D-13, D-15.)
- Background: #111114. Border-radius: 12px.
- Custom drag region: top 48px.
- No close button — user exits only via "Get started" or "Continue anyway".
  (Prevents accidental dismissal before user sees results.)
- Source list rows: same layout as companion `.achievement-row` minus the left accent bar.
  Padding 8px, border-radius 8px, background #1C1C21.
  Found row: ✓ in accent rgba(120,220,255,0.85) at 14px/600.
  Not-found row: ✗ in rgba(240,240,245,0.55) at 14px/400.

### Update Modal

- Rendered as a modal sheet layered over the companion window (not a separate OS window).
- Backdrop: rgba(0,0,0,0.55) full-companion overlay.
- Card: background #1C1C21, border-radius 12px, padding 24px,
  max-width 400px, centered in companion.
- Enter animation: fade-in + scale 0.96→1.0, 200ms ease-out (Framer Motion).
- Exit animation: fade-out + scale 1.0→0.96, 150ms ease-in.

### Tray Menu

- Rendered by OS/Tauri native system-tray subsystem. No custom CSS.
- Structure is locked per CONTEXT.md D-01:
  ```
  Hallmark
  ─────────────
  Show companion
  Fire test popup
  ─────────────
  Settings…
  ☑ Start with Windows
  ─────────────
  Quit
  ```
- "☑ Start with Windows" is an inline checkable menu item — state reflects live registry
  (HKCU\Software\Microsoft\Windows\CurrentVersion\Run). Read on menu open, not cached.
- Tray icon: monochrome glyph (single-color SVG, 16×16 or 32×32 for high-DPI).
  Recommended design: a simple trophy/medal outline glyph, rendered white on dark
  taskbar or black on light taskbar. Tauri's `TrayIconBuilder` accepts separate
  `icon` + `icon_as_template` for light/dark tray variants.

### Error states

| Surface | Trigger | Copy |
|---------|---------|------|
| Settings — Rescan failed | discover_paths returns error | Something went wrong during rescan. Try again. |
| Settings — Update check failed | Network unreachable or GitHub timeout | Couldn't reach the update server. Check your connection. |
| Settings — Update check in-progress | Awaiting GitHub response | Checking for updates… |
| Wizard — Rescan failed | discover_paths returns error | Scan failed. You can continue and rescan from Settings later. |
| Update modal — Download failed | tauri-plugin-updater download error | Download failed. Check your connection and try again. |

### Destructive actions

| Action | Surface | Confirmation approach |
|--------|---------|----------------------|
| Quit | Tray menu — "Quit" item | No confirmation dialog. Immediate clean shutdown. Drain popup queue with timeout. |
| Toggle off "Start with Windows" | Tray menu checkbox toggle | No confirmation. Registry value removed immediately. Tray item reflects new state. |

No destructive actions require an explicit confirmation dialog in Phase 4. Quit and
autostart-off are low-stakes and reversible by the user.

## Interaction States

### Settings — Detected Sources panel

| State | Trigger | Visual treatment |
|-------|---------|-----------------|
| Initial load | Settings opens | Show discovered sources from last discover_paths run (cached in memory). |
| Scanning | "Rescan" clicked | Button label → "Scanning…"; source rows replaced with 3 skeleton lines (same `.skeleton-line` animation as companion). |
| Scan complete | discover_paths resolves | Rows update. ✓ accent / ✗ muted. Button label → "Rescan". |
| Scan error | discover_paths rejects | Rows stay unchanged. Error text appears below button (14px/400, text-secondary). |

### Settings — Updates panel

| State | Trigger | Visual treatment |
|-------|---------|-----------------|
| Idle | Settings opens | Show current version + last-checked timestamp. "Check for Updates" button enabled. |
| Checking | Button clicked | Button label → "Checking for updates…", disabled. |
| Up to date | latest.json version ≤ current | "You're on the latest version." 14px/400 text-secondary. Timestamp updates to "just now". |
| Update available | latest.json version > current | "Version {x.y.z} is available." 14px/400 in text-primary. Version badge in accent. "Install and Restart Hallmark" button replaces check button. |
| Check failed | Network error | Error copy below button. Button re-enabled as "Retry". |

### First-run wizard — "Get started" vs. "Continue anyway"

- "Get started" is the primary CTA. Rendered as a pill button: background #1C1C21,
  border 1px solid rgba(120,220,255,0.40) (subtle accent border), text #F0F0F5 14px/600.
  This is the ONLY button with an accent border in the wizard — it signals the positive path.
- "Continue anyway" is a secondary text-button: no background, no border, text
  rgba(240,240,245,0.55) 14px/400. Positioned below "Get started".
- Both appear only when N=0. When N>0, only "Get started" is shown (no "Continue anyway").

### Autostart toggle state sync

- The "☑ Start with Windows" tray menu item reflects the live HKCU registry state.
- Registry is read each time the tray menu opens (not cached in memory across sessions).
- Toggle is instant — no loading state shown in tray menu (registry write is synchronous,
  <1ms on Windows). No confirmation or undo surface needed.
