# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Premium UI — Popup, Companion & Game Session.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual CSS |
| Preset | not applicable |
| Component library | none — hand-rolled React components |
| Icon library | none — achievement icons are fetched from Steam appcache/CDN; UI icons use CSS shapes or inline SVG |
| Font | Inter (system fallback chain — see Typography section) |

**shadcn gate decision:** shadcn was evaluated and declined for this phase. The "PS5 Pure" dark glass material direction (near-black backdrop-blur surface, luminous cyan accent stroke, white text) is incompatible with shadcn's default design tokens and would require full override of every token. The popup and companion are also non-standard surfaces (one non-interactive overlay, one borderless card) with no standard form controls in scope. Manual CSS gives exact control of the backdrop-filter, border-glow, and spring animation without fighting a component library's opinions. Registry safety gate: not applicable.

Source of design decisions: CONTEXT.md (D-01..D-28), RESEARCH.md (Pattern 5 spring values, popup.css direction), REQUIREMENTS.md (POPUP-01..08, COMP-01..03).

## Surfaces

This phase produces exactly two UI surfaces. Both are separate Tauri WebviewWindows with `decorations: false`.

| Surface | Label | Size (logical px) | On-top | Interactive |
|---------|-------|-------------------|--------|-------------|
| Popup overlay | `popup` | 440 × 96 | always-on-top (`WS_EX_TOPMOST`) + `WS_EX_NOACTIVATE` | no — click-through via `set_ignore_cursor_events(true)` |
| Companion window | `companion` | 480 × 720 (default, user-resizable, persisted) | normal window | yes |

Source: CONTEXT.md D-02, D-13, D-14. RESEARCH.md Pattern 1 (popup builder), Pattern 5 (Framer Motion tree).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside popup pill; badge inner padding |
| sm | 8px | Popup pill internal side padding (icon left margin); companion row horizontal padding |
| md | 16px | Companion window internal horizontal padding; popup pill vertical padding |
| lg | 24px | Companion section breaks (between filter bar and list) |
| xl | 32px | Screen-edge margin for popup placement (`margin` constant in `popup_position()`); companion window internal top padding |
| 2xl | 48px | Companion header block height |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Popup pill border-radius: 48px (full-pill stadium shape — D-02). This is a border-radius, not a spacing token.
- Popup icon disc: 56px diameter (fits within 96px pill height with 20px vertical padding each side).
- Companion row expand area: min-height 72px when expanded (timestamp + full description).
- Touch/click targets in companion: minimum 44px height per row (accessibility minimum for interactive rows per WCAG 2.5.5).

Source: CONTEXT.md D-02, D-14; RESEARCH.md popup window `inner_size(440.0, 96.0)`; REQUIREMENTS.md POPUP-01, COMP-02.

## Typography

Font stack (no licensed dependency — system font first, premium fallback second):

```
font-family: "Inter", "Segoe UI Variable", "Segoe UI", -apple-system, BlinkMacSystemFont, system-ui, sans-serif;
```

Inter is available on most modern Windows 11 machines and from a CDN via Vite asset pipeline if needed. `Segoe UI Variable` is the Windows 11 system variable font. If neither is available, system-ui covers the rest. No web font license required.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Popup title | 14px | 600 (semibold) | 1.2 | Achievement display name in popup pill (D-02 "title bold") |
| Popup description | 12px | 400 (regular) | 1.4 | Achievement description in popup pill — truncated to 2 lines max; also used for rarity tag ("X.X% of players", POPUP-07) |
| Companion heading | 16px | 600 (semibold) | 1.2 | Game name in companion header; section labels |
| Companion body | 14px | 400 (regular) | 1.5 | Achievement title in companion list rows (COMP-02); filter chip labels; sort toggle labels |
| Companion meta | 12px | 400 (regular) | 1.4 | Unlock timestamp + description in expanded row |

Declared weights: **400 (regular)** and **600 (semibold)**. No other weights used.
Declared sizes: **12, 14, 16** — three sizes spanning both surface contexts. Popup title and companion body share 14px (both are primary row labels in their respective surfaces). Popup description and companion meta share 12px (both are secondary/supporting text). The rarity tag previously at 11px is merged into 12px — at gaming viewing distances, the distinction was sub-perceptual. The former 13px companion body is promoted to 14px to align with the popup title and eliminate the 1px-gap hierarchy problem.

Source: CONTEXT.md D-02 ("title bold + description"), D-18 (filter/sort state); RESEARCH.md Pattern 5 (`popup-title`, `popup-desc`, `popup-rarity` class names). Discretion applied for exact sizes and font stack per CONTEXT.md "Claude's Discretion." Revised to resolve Typography BLOCK (5 sizes → 3 sizes) and FLAG (1px-gap sizes eliminated).

### Companion Surface

| Role | Value | Usage |

| Destructive | `#E05252` | Close button hover state only — no destructive actions in this phase |

### Rare-Tier Modifier (POPUP-06)
When `tier === "rare"` (global unlock rate < 10% with rarity data available):
- Pill outer border stroke upgrades from `rgba(120, 220, 255, 0.85)` to `rgba(180, 255, 255, 1.0)` (brighter)
- Icon disc gains a `box-shadow: 0 0 12px 3px rgba(120, 220, 255, 0.55)` halo ring
- Popup background opacity increases from 0.82 to 0.88 (slightly more opaque — the moment deserves presence)
- No additional color is introduced — only intensity is elevated

### 100% Celebration Modifier (POPUP-05, D-12)
When `tier === "completion"`:
- Same rare-tier styling PLUS a brief `scale(1.04) → scale(1.0)` pulse on entry (separate Framer Motion keyframe)
- Background: `rgba(20, 14, 30, 0.90)` — warmer dark purple undertone to differentiate from standard rare
- Accent stroke: `rgba(220, 180, 255, 0.90)` — shifts to cool purple for the celebration moment

Accent reserved for: popup outer border stroke, rare-tier icon halo, companion filter chip active indicator, companion earned-row left stripe, session unlock count badge.

Source: CONTEXT.md D-03 ("dark glass with translucent backdrop-blur, cool white/cyan accent stroke, white text"), D-06 (rare-tier), D-12 (celebration), D-07 (degrade when no rarity data). Discretion applied for exact hex values and companion color palette per CONTEXT.md "Claude's Discretion."

## Animation Contract

Source: CONTEXT.md D-04, D-08, D-09, D-10; RESEARCH.md Pattern 5 (spring values, Framer Motion choreography).

### Popup Entry / Exit (Framer Motion spring)

```
Framer Motion spring: { type: "spring", stiffness: 380, damping: 28, mass: 0.9 }
Entry:  { x: 480, opacity: 0 } → { x: 0, opacity: 1 }   ~300ms perceived settle
Exit:   { x: 0, opacity: 1 }  → { x: 0, y: -16, opacity: 0 }  ~300ms
```

- `useReducedMotion()` checked; if true, entry/exit uses `{ opacity: 0/1 }` at `duration: 0.15` only (no positional movement).
- `AnimatePresence` wraps the pill component; `key={ach_api_name}` forces re-mount on each new unlock.

### Popup Hold Timing

| Condition | Slide-in | Hold | Slide-out | Inter-popup gap |
|-----------|----------|------|-----------|-----------------|
| Queue depth ≤ 5 | 300ms | 3000ms | 300ms | 200ms |
| Queue depth > 5 (burst) | 300ms | 1500ms | 300ms | 0ms |

Source: CONTEXT.md D-08, D-09, D-10. Timing is enforced in Rust (`popup_queue.rs` sleep calls); React only handles visual in/out.

### Companion List

- New earned entries animate in with `opacity: 0 → 1` over 200ms when schema resolves and upgrades an `api_name` placeholder to `display_name` (D-20 in-place upgrade).
- Row expand/collapse: `height: auto` via Framer Motion `layout` prop + `overflow: hidden` clip; 150ms ease.
- Filter chip state change: instant (no animation — filter is a data concern, not a spatial one).

## Icon Treatment

Source: CONTEXT.md "Claude's Discretion" (icon framing, shape inside pill).

| Context | Shape | Size | Treatment |
|---------|-------|------|-----------|
| Popup icon disc | Circular | 56px diameter | `border-radius: 50%`; `object-fit: cover`; 2px gap from pill left edge. Standard tier: no ring. Rare tier: 3px cyan halo via `box-shadow`. |
| Popup icon placeholder (D-26) | Circular | 56px diameter | Solid `rgba(255,255,255,0.15)` fill with no image. No "?" text — empty silence is cleaner than a broken fallback label. |
| Companion row icon | Circular | 36px diameter | `border-radius: 50%`; `object-fit: cover`. Locked icons: `opacity: 0.4`. |
| Companion row icon placeholder | Circular | 36px diameter | `rgba(255,255,255,0.10)` fill. |

Decision: circular over squircle. Achievement icon artwork is square with round-corner-friendly content, so circular clips cleanly. Squircle (CSS `clip-path: path()`) adds complexity for minimal gain on this icon size.

## DPI and Scaling

Source: REQUIREMENTS.md POPUP-04; RESEARCH.md Pattern 2 (DPI handling note).

- All CSS values in this spec are **logical pixels**. Tauri/TAO uses per-monitor DPI awareness (Windows 10 1703+ `DPI_AWARENESS_CONTEXT_PER_MONITOR_AWARE_V2` — automatic). WebView2 scales CSS pixels by `devicePixelRatio` automatically.
- The popup `inner_size(440.0, 96.0)` in Tauri is in logical pixels — renders as 880×192 physical pixels on a 2× DPI display.
- The `popup_position()` calculation in `monitor.rs` works in physical pixels (as returned by `GetMonitorInfoW`); it uses `PhysicalPosition` for Tauri placement. This is correct and DPI-transparent.
- No `transform: scale()` or `zoom` CSS is needed. Trust the browser's DPI scaling.
- Minimum readable text on 4K: 12px logical = 24px physical on 2× = legible at typical gaming distance.

## Accessibility

- `useReducedMotion()` from Framer Motion is checked in `PopupRoot`. All positional animations are replaced with opacity-only fade at `duration: 0.15` when the system accessibility setting is on. (WCAG 2.3.3 Animation from Interactions — Level AAA; implemented as best-effort in a game overlay context.)
- Close button has `aria-label="Close companion"`.
- Filter chips use `role="radio"` + `aria-checked` for screen reader state.
- Achievement rows use `role="listitem"` within a `role="list"`.
- All non-decorative images (achievement icons) have `alt=""` in the popup (decorative context — the title text is the primary label) and `alt="{display_name} achievement icon"` in the companion list.
- The popup overlay is entirely non-interactive (`WS_EX_NOACTIVATE` + `set_ignore_cursor_events(true)`) and does not appear in the accessibility tree. This is intentional — it is a transient ambient notification, not actionable UI.
